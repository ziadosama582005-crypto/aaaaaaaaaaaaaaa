# 📋 دليل الاختبار السريع

## 🧪 خطوات الاختبار

### 1. التحقق من التثبيت
```bash
cd /workspaces/aaaaaaaaaaaaaaa
python3 -m pip install -r requirements.txt
```

### 2. تشغيل التطبيق
```bash
python3 app.py
```

### 3. فتح المتصفح
```
http://localhost:5000
```

## 🧬 اختبار الميزات

### A. اختبار نظام المستخدمين
```
1. الذهاب إلى /login
2. إدخال الهاتف
3. الحصول على OTP
4. إدخال الكود (يظهر في الـ console)
5. تأكيد التسجيل
```

### B. اختبار صفحة الفئات
```
1. الذهاب إلى /categories
2. اختيار أي فئة
3. يجب أن تظهر المنتجات المتاحة
4. فلترة حسب: متاح، مباع، مشترياتي
```

### C. اختبار السلة
```
1. من صفحة الفئة → اختيار منتج
2. النقر على "إضافة للسلة"
3. الذهاب إلى /cart
4. يجب أن تظهر البيانات الجديدة
5. محاولة حذف عنصر
```

### D. اختبار الشراء
```
1. في صفحة السلة
2. النقر على "إتمام عملية الشراء"
3. يجب التحقق من الرصيد كافٍ
4. سيتم الانتقال إلى /my_purchases
```

### E. اختبار لوحة التحكم
```
1. الدخول بحساب admin (user_id = 1)
2. الذهاب إلى /admin/dashboard
3. عرض الإحصائيات
4. إدارة المنتجات: /admin/products
5. عرض الطلبات: /admin/orders
6. إدارة المستخدمين: /admin/users
7. التقارير: /admin/reports
```

### F. اختبار API مباشرة
```bash
# عرض السلة
curl http://localhost:5000/api/cart/view

# إضافة للسلة
curl -X POST http://localhost:5000/api/cart/add \
  -H "Content-Type: application/json" \
  -d '{"product_id": "123", "buyer_details": ""}'

# عرض الإحصائيات
curl http://localhost:5000/admin/api/stats
```

## 🧪 حالات الاختبار الخاصة

### حالة 1: منتج مباع
- المنتج يظهر في تبويب "المباعة"
- لا يمكن إضافته للسلة
- يظهر مع شارة "✓ مباع"

### حالة 2: منتج تسليم يدوي
- يظهر حقل "تفاصيل المشتري" في صفحة المنتج
- يتم حفظ البيانات في قاعدة البيانات
- يظهرها الأدمن في صفحة الطلب

### حالة 3: شراء بدون رصيد كافٍ
- يظهر خطأ عند محاولة الشراء
- لا يتم تنفيذ الشراء
- يطلب المستخدم شحن الرصيد

### حالة 4: سلة فارغة
- عند الدخول للسلة بدون منتجات
- يظهر رسالة "السلة فارغة"
- يظهر زر "ابدأ التسوق الآن"

## 🐛 تتبع الأخطاء

### الأخطاء الشائعة والحل

1. **خطأ: "Firebase not initialized"**
   - تأكد من تعيين متغيرات البيئة
   - تحقق من ملف `credentials.json`

2. **خطأ: "Collection not found"**
   - تأكد من إنشاء Collections في Firebase
   - استخدم script إنشاء البيانات الأولية

3. **خطأ: "Session expired"**
   - قم بتسجيل الدخول مجدداً
   - امسح cookies المتصفح

4. **خطأ: "API timeout"**
   - تحقق من اتصال الإنترنت
   - تحقق من حالة Firebase

## 📈 مؤشرات الأداء

- وقت تحميل الصفحة: < 2 ثانية
- سرعة استجابة API: < 500ms
- معدل الأخطاء: < 1%
